# FAQ・トラブルシューティング

## セクション 1：よくある質問（FAQ）

### PoC LP・サンプルレポート全般

#### Q1. サンプルレポートのデータは実在するデータですか？
**A：** いいえ。サンプルレポートは、**仮想データ**を基に構成しています。業界データ、既存顧客実績の匿名化データ、統計的な予測を組み合わせて作成しており、実在する顧客データではありません。

実装時（Phase 7 以降）は、顧客の実績データを用いて、このテンプレートに置き換える予定です。

---

#### Q2. レポートはどのくらいの頻度で更新されますか？
**A：** 毎月末日時点での過去 1 ヶ月間のデータを集計し、翌月初（通常 1～3 営業日以内）に新規レポートを生成することを想定しています。

リアルタイム更新ではありませんが、営業チームのデータドリブン意思決定に必要な頻度で提供される設計です。

---

#### Q3. HTML 版と Markdown 版の違いは何ですか？
**A：** 

| 項目 | HTML 版 | Markdown 版 |
|------|--------|-----------|
| 表示方法 | ブラウザで直接表示 | テキストエディタで編集・表示 |
| 装飾・スタイル | CSS で自動的に適用 | プレーンテキスト（簡素） |
| 編集難易度 | 難しい（要 HTML 知識） | 簡単（テキストエディタで可） |
| 共有方法 | URL リンク、メール添付 | メール本文、チャットペースト可 |
| 用途 | クライアント提示、プレゼン | 内部議論、ドキュメント管理 |

**選択のポイント：**
- 顧客に見せる → HTML 版
- 社内で議論・編集する → Markdown 版

---

#### Q4. レポートのメトリクス（指標）はカスタマイズできますか？
**A：** はい。以下の項目は柔軟にカスタマイズ可能です：

- **可能：** テーマ分類、成長テーマの件数（Top 3 → Top 5 など）、推奨アクション内容、期待効果の数値
- **一部可能：** エンゲージメント率の算出ロジック（有償オプション）
- **難しい：** 基本フレーム（Executive Summary、テーマ別、講座別の 3 軸分析）

詳細なカスタマイズについてはサポートまでお問い合わせください。

---

#### Q5. 「推奨アクション」の根拠は何ですか？
**A：** 以下の複合的な分析から生成されます：

1. **成長テーマ分析**：前月比で視聴数が増加しているテーマを特定
2. **完了率分析**：視聴開始後、最後まで見られている講座を優先
3. **転換率分析**：ある講座を見た後、購買に至りやすい次の講座
4. **業界ベンチマーク**：EdTech 業界の一般的な成功施策
5. **営業チームのドメイン知識**：実際の顧客成功事例

つまり、統計的根拠 + 定性的経験の融合です。

---

#### Q6. 期待効果の数値（「10～15% 程度の上昇」など）はどうやって算出しましたか？
**A：** 保守的な推定を心がけています：

- **成長テーマへの投資** → 業界平均 15～22% の視聴数増を見込めるが、実装品質に左右されるため、**10～15% に控えめに設定**
- **既存講座の統合** → 相乗効果で 5～10% の追加成長を期待
- **期待効果の金額表記**（「月数十～数百万円」など） → 講座数・価格帯によって幅が大きいため、範囲で示記

実装後の実績との検証が重要です。

---

#### Q7. 「レポート作成者note」とは何ですか？
**A：** レポート生成時点での注記です。内容は以下の通り：

> 本サンプルレポートは仮想データを基に構成。Phase 7 以降は匿名化した実績データに置換予定。

つまり、**「このレポートはサンプルである」という免責事項**です。顧客に提示する場合は、必ず「サンプル版です」と明示してください。

---

### データ・メトリクス関連

#### Q8. 「エンゲージメント率」の定義は何ですか？
**A：** video-insight-spec 独自の複合指標です。以下の要素を加重平均して算出しています：

- **視聴完了率**：動画の最後まで見た割合（30% の重み）
- **リプレイ率**：動画を複数回視聴した割合（25% の重み）
- **課題提出率**：講座に付属する課題を提出した割合（25% の重み）
- **コミュニティ参加率**：ディスカッション、Q&A に参加した割合（20% の重み）

**目安（この数値より高いと優良コンテンツ）：**
- 1.5% 以上：優秀
- 1.0～1.5%：標準的
- 1.0% 未満：改善推奨

---

#### Q9. 「前月比」が負の数になることはありますか？
**A：** はい。可能性があります。例えば：

- 季節性（夏休み時期に学習量が減少）
- 競合サービスの登場
- コンテンツの老化（同じテーマでも新作のほうが選ばれやすい）

負の数が続く場合は、レポートの「インサイト」セクションで原因分析と改善策を提案します。

---

#### Q10. 講座数が少ない（1～2 本）場合、このテンプレートは使えますか？
**A：** 部分的に使えます：

- **使える部分**：Executive Summary（全体数値）、テーマ別分析（該当テーマのみ）
- **難しい部分**：「成長テーマ Top 3」（テーマ数が 3 未満の場合）、「Top 3 講座」（講座数が 3 未満の場合）

この場合、セクションを簡略化した「ライト版」テンプレートの利用を検討してください（別途提供予定）。

#### Q13. 既存のレポート作成ツール（Excel、BIツール など）からの移行は簡単ですか？
**A：** はい。以下のメリットがあります：

- **作業時間の削減**：従来 12 時間 → 2～3 時間（80% 削減）
- **ヒューマンエラーの低減**：自動計算なので集計ミスがない
- **営業メッセージの自動生成**：推奨アクションがテンプレートから即座に提案される

ただし、既存ツールの独自カスタムを多く行っていた場合は、確認が必要です。

**期待値の設定：**
- 楽観的すぎず、実現可能な数値を示す
- 「貴社の業種・規模では、月 ¥250,000～¥375,000 の売上インパクトが期待できます」など定量化

### セキュリティ・プライバシー関連

#### Q16. 顧客データのセキュリティは大丈夫ですか？
**A：** はい。以下の対策を実施しています：

- **暗号化**：転送時（HTTPS）、保存時（AES-256）
- **アクセス制限**：IP ホワイトリスト、ロールベースアクセス制御
- **監査ログ**：全アクセスを記録し、不正利用を検知
- **コンプライアンス**：GDPR、個人情報保護法対応
- **定期監査**：第三者セキュリティ会社による年 1 回の監査

詳細はセキュリティホワイトペーパーをご参照ください。

---

#### Q17. レポートに含まれる「顧客の講座情報」は、他の顧客に見られることはありませんか？
**A：** いいえ。各顧客のデータは完全に隔離されています：

- **ログイン認証**：顧客専用の ID・パスワードで保護
- **テナント分離**：顧客 A のデータは顧客 B からアクセス不可
- **監査ログ**：誰がいつアクセスしたかを記録

また、video-insight-spec チーム内でも、必要最小限の担当者のみがアクセス権を持ちます。

## セクション 2：トラブルシューティング

### HTML ファイルが表示されない場合

**症状：** ブラウザで HTML ファイルを開いても、白い画面またはエラーメッセージが表示される

**対応ステップ：**
1. **ブラウザキャッシュをクリア**
   - Chrome: Ctrl+Shift+Delete → キャッシュをクリア
   - Safari: メニュー → 履歴 → 履歴を消去

2. **別のブラウザで試す**
   - Chrome、Firefox、Safari など複数試す

3. **ファイルのパスを確認**
   - ファイルがダウンロードフォルダ内に正しく保存されているか確認
   - パスに日本語が含まれていないか確認（問題がある場合は英数字のみのフォルダに移動）

4. **HTML ファイルのサイズを確認**
   - ファイルサイズが 50KB 以上あるか確認（破損していない）
   - メール添付で圧縮されていないか確認

5. **サポートに連絡**
   - 上記で解決しない場合、support@video-insight-spec.example.com までご連絡ください
   - ファイル名、エラーメッセージ、使用ブラウザ・OS をお知らせください

---

### レポートの数値が正しく表示されない場合

**症状：** 表やグラフに「エラー」「NaN」「0」などが表示される

**対応ステップ：**
1. **データソースを確認**
   - データが正しく接続されているか確認（管理画面で確認）
   - データの形式が合っているか確認（例：金額は数値型か）

2. **テンプレートをリセット**
   - 古いキャッシュが残っていないか確認
   - テンプレートを再度ダウンロード・適用

3. **データを再インポート**
   - CSV や Excel からのインポート時に エンコーディング（UTF-8）を確認
   - 最初の 10 行を手動で確認し、形式を検証

4. **サポートに連絡**
   - 該当するデータサンプル（個人情報を削除した上で）をお送りください

---

### メール送信時にレポートが添付されない場合

**症状：** 「メールを送信」ボタンをクリックしても、メールが送られない or 添付ファイルがない

**対応ステップ：**
1. **メール設定を確認**
   - メールアプリ（Outlook、Gmail など）が正しく設定されているか確認
   - メーラーのポップアップブロック設定を確認

2. **ファイルサイズを確認**
   - HTML ファイルが 50MB を超えていないか確認（大きい場合は圧縮が必要）

3. **ブラウザのセキュリティ設定**
   - ブラウザが自動ダウンロードをブロックしていないか確認
   - 許可されたサイト リストに video-insight-spec ドメインを追加

4. **サポートに連絡**
   - 使用しているメールアプリ、ブラウザ、OS をお知らせください

---

### レポート内の日本語が文字化けしている場合

**症状：** レポート内の日本語が四角い記号「□」に見える

**対応ステップ：**
1. **ブラウザの文字エンコーディングを確認**
   - ブラウザのメニュー → 文字エンコーディング → UTF-8 に変更

2. **OS のロケール設定を確認**
   - Windows：コントロールパネル → 地域と言語 → 日本語に設定
   - Mac：システム環境設定 → 言語と地域 → 日本語を最優先に設定

3. **HTML ファイル自体の文字コードを確認**
   - テキストエディタで開き、「文字コード → UTF-8 with BOM」で保存し直す

4. **サポートに連絡**
   - 上記で解決しない場合、お気軽にお問い合わせください
